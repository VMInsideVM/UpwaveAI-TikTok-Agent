# 商品分类放宽功能说明

## 功能概述

当达人数量不足时，Agent 现在可以建议将商品分类放宽到上一级，以扩大候选范围。这是**唯一允许修改商品分类**的情况。

---

## 功能特点

### ✅ 智能层级放宽

- **L3 → L2**: 三级分类放宽到二级分类（预计增加 40-60%）
- **L2 → L1**: 二级分类放宽到一级分类（预计增加 60-100%）
- **L1**: 已经是最高级，无法继续放宽

### ✅ 自动查找父分类

系统会自动：
1. 读取对应的分类 JSON 文件
2. 查找当前分类的上一级
3. 获取父分类的 ID 和 URL 后缀
4. 生成完整的调整方案

### ✅ 用户确认机制

- 分类放宽方案会出现在调整建议列表中（优先级 2）
- 用户必须**明确选择**该方案才会执行
- 不会自动修改分类

## 使用场景

### 场景 1: 精确分类导致达人太少

**用户输入**:
```
商品: 餐盘
国家: 美国
粉丝: 20万-50万
需要: 50 个达人
```

**Agent 行为**:
```
1. 匹配到 L3 分类"盘"（category_id=600064）
2. 检查数量: 最大页数 3，约 30 个达人
3. 判断: 数量不足（30 < 50）
4. 生成调整建议:

📊 为您生成 2 个调整方案：

**方案 1: 放宽粉丝数范围**
  • 当前: 200,000 - 500,000
  • 调整后: 100,000 - 1,500,000
  • 预期效果: 预计增加 100-150%

**方案 2: 放宽商品分类范围**
  • 当前: L3 分类: 盘
  • 调整后: L2 分类: 餐具
  • 预期效果: 预计增加 40-60%
  • 理由: 将分类从三级"盘"放宽到二级"餐具"，可包含更多相关达人
```

**用户选择方案 2 后**:
```
✅ 已应用方案 2：放宽商品分类范围
   从"盘"(L3) → "餐具"(L2)

现在搜索范围包括：
- 盘（原分类）
- 碗
- 筷子
- 叉子
- 勺子
- ... 所有餐具类达人

重新检查数量: 最大页数 12，约 120 个达人
✅ 数量充足！
```

---

### 场景 2: 二级分类也不够，继续放宽

如果用户再次遇到数量不足，可以继续放宽到 L1：

```
**方案 2: 放宽商品分类范围**
  • 当前: L2 分类: 餐具
  • 调整后: L1 分类: 厨房用品
  • 预期效果: 预计增加 60-100%
  • 理由: 将分类从二级"餐具"放宽到一级"厨房用品"，显著扩大候选范围
```

应用后，搜索范围扩大到整个厨房用品类：
- 餐具（原分类）
- 厨房器具
- 饮具
- 炊具
- ... 所有厨房相关达人

## 技术实现

### 1. 新增函数: `get_parent_category_url()`

**文件**: [adjustment_helper.py](adjustment_helper.py)

**功能**: 根据当前分类信息，自动查找上一级分类

**输入**:
```python
category_info = {
    'level': 'l3',
    'category_name': '盘',
    'main_category': '厨房用品',
    'url_suffix': '&category_id=600064'
}
```

**输出**:
```python
{
    'level': 'l2',
    'category_name': '餐具',
    'url_suffix': '&category_id=859272',
    'main_category': '厨房用品'
}
```

**算法**:
1. 读取 `categories/{main_category}.json`
2. **L3 → L2**: 遍历所有 L2 分类，查找包含当前 L3 的 L2
3. **L2 → L1**: 直接使用主分类（L1）ID

---

### 2. 更新 `suggest_adjustments()` 函数

**优先级调整**:
```
旧版本:                          新版本:
1. 放宽粉丝数范围                 1. 放宽粉丝数范围
2. 移除新增粉丝限制         →     2. 放宽商品分类范围 ⭐ (新增)
3. 移除联盟达人限制               3. 移除新增粉丝限制
4. 移除认证类型限制               4. 移除联盟达人限制
5. 移除账号类型限制               5. 移除认证类型限制
                                  6. 移除账号类型限制
```

**新增逻辑**:
```python
# 检查是否可以放宽分类
category_info = current_params.get('category_info')
if category_info:
    parent_category = get_parent_category_url(category_info)

    if parent_category:
        # 生成分类放宽方案
        suggestions.append({
            'priority': 2,
            'name': '放宽商品分类范围',
            'changes': {
                '_parent_category': parent_category  # 存储完整的父分类信息
            },
            'current': f'{current_level} 分类: {current_category}',
            'new': f'{parent_level} 分类: {parent_name}',
            'expected_increase': expected_increase,
            'reason': reason
        })
```

### 3. Agent 提示词更新

**文件**: [agent.py](agent.py) - 第 186-197 行

**新增执行逻辑**:
```
9. **执行调整** (如果用户选择了调整方案):
   - **特殊处理 - 分类放宽**:
     * 如果方案包含 '_parent_category'，说明需要放宽商品分类
     * 从 changes['_parent_category'] 获取上级分类信息
     * 更新 agent.current_params['category_info'] 为父分类
     * 使用父分类的 url_suffix 替换原分类后缀
   - **常规参数调整**:
     * 应用新的筛选参数（粉丝数、认证类型等）
     * 保持国家不变
   - 重新构建 URL（使用新的分类后缀，如果分类被放宽）
   - 重新检查数量 (回到步骤6)
```

**更新的规则**:
```
- 商品分类**默认不能修改**，但有一个例外：
  * 当达人数量严重不足时，可以通过调整方案将分类放宽到上一级（L3→L2 或 L2→L1）
  * 这是唯一允许修改分类的情况，且必须通过 suggest_parameter_adjustments 工具生成方案
  * 用户必须明确选择该方案才能执行
```

## 使用限制

### ❌ 不允许的操作

1. **不能跨级放宽**: 不能从 L3 直接跳到 L1（必须先 L3→L2，再 L2→L1）
2. **不能缩小范围**: 不能从 L1 缩小到 L2 或 L3
3. **不能手动指定分类**: 用户不能说"把分类改成 XX"
4. **不能在数量充足时放宽**: 只有数量不足时才会生成此方案

### ✅ 允许的操作

1. **数量不足时放宽**: 系统自动生成放宽建议
2. **用户明确选择**: 用户选择"方案 2"来应用分类放宽
3. **多次放宽**: 可以先 L3→L2，如果还不够，再 L2→L1
4. **与其他调整组合**: 可以同时放宽分类和粉丝数范围

## 预期效果估算

基于分类层级结构和历史数据：

| 放宽方式 | 预期增加 | 原因 |
|---------|---------|------|
| **L3 → L2** | 40-60% | L2 通常包含 5-10 个 L3 子分类 |
| **L2 → L1** | 60-100% | L1 通常包含 3-8 个 L2 子分类 |

**实际效果可能因分类而异**:
- 热门分类（如"手机壳"）: 放宽效果较小（~30%）
- 小众分类（如"冰淇淋工具"）: 放宽效果较大（~80%）

## 优势与劣势

### ✅ 优势

1. **扩大候选池**: 显著增加可选达人数量
2. **语义相关性**: 放宽后仍保持一定的相关性（如盘→餐具）
3. **用户控制**: 需要用户明确选择，不会强制修改
4. **渐进式放宽**: 支持多次放宽（L3→L2→L1）

### ⚠️ 劣势

1. **相关性降低**: 放宽后可能包含不太相关的达人
2. **筛选工作增加**: 用户需要从更大的候选池中筛选
3. **精准度下降**: 从精确分类变为宽泛分类

### 💡 建议

- **优先尝试其他调整**: 先放宽粉丝数范围，如果还不够再考虑放宽分类
- **告知用户权衡**: 向用户说明放宽分类后，相关性会降低，但数量会增加
- **提供组合方案**: "建议同时放宽粉丝数和分类，效果最佳"
