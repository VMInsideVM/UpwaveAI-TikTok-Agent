# 登录功能说明文档

## 概述

Playwright API 现在支持自动登录检查功能。当访问 FastMoss 网站时，系统会自动检测登录状态，如果未登录则自动执行登录流程。

## 功能特性

### ✅ 自动化登录
- 无需手动登录
- 系统自动检测登录状态
- 未登录时自动执行登录流程

### ✅ 智能缓存
- 登录状态缓存 5 分钟
- 避免频繁检查，提升性能
- 自动更新缓存状态

### ✅ 并发安全
- 使用 `asyncio.Lock` 防止并发登录
- 同一时间只允许一个登录流程执行
- 其他请求会等待登录完成

### ✅ 失败重试
- 登录失败自动重试（默认 1 次）
- 智能等待策略（2 秒间隔）
- 清晰的错误信息

### ✅ 详细日志
- 每个步骤都有日志输出
- 使用 emoji 图标便于识别
- 方便调试和监控

## 登录流程

系统会执行以下 9 个步骤来完成登录：

```
1️⃣ 检查登录状态
   └─ 查找 "登录/注册" 按钮
      ├─ 未找到 → 已登录 ✅
      └─ 找到 → 未登录，继续登录流程

2️⃣ 点击 "登录/注册" 按钮

3️⃣ 点击 "手机号登录/注册" 选项

4️⃣ 输入手机号
   └─ 13337315183

5️⃣ 切换到 "密码登录"

6️⃣ 输入密码
   └─ julian13579

7️⃣ 点击 "注册/登录" 按钮

8️⃣ 等待登录完成 (2秒)

9️⃣ 验证登录结果
   └─ 检查登录按钮是否消失
```

## 使用方法

### 无需任何额外操作！

只需正常使用 API，系统会自动处理登录：

```python
import requests

# 1. 导航到搜索页面（自动登录）
response = requests.post(
    "http://127.0.0.1:8000/navigate",
    json={"url": "https://www.fastmoss.com/zh/influencer/search"}
)

# 2. 获取最大页数（使用缓存，无需重复登录）
response = requests.get("http://127.0.0.1:8000/max_page")

# 3. 爬取数据（使用缓存）
response = requests.post(
    "http://127.0.0.1:8000/scrape",
    json={
        "urls": ["https://www.fastmoss.com/zh/influencer/search?region=US"],
        "max_pages": 5,
        "product_name": "测试商品"
    }
)
```

## 错误处理

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| `200` | 成功（已登录或登录成功） |
| `401` | 登录失败（凭证错误或达到重试上限） |
| `500` | 技术错误（元素未找到、超时等） |
| `503` | Playwright 未初始化 |

### 错误响应示例

```json
{
  "detail": "登录失败: 自动登录失败: 达到最大重试次数"
}
```

    "phone": "13337315183",

**生产环境建议**：将凭证移到 `.env` 文件中：

```env

```

### 缓存配置

登录状态缓存 5 分钟，定义在 `playwright_api.py` 第 243 行：

```python
if last_check and (datetime.now() - last_check).seconds < 300:
```

可以根据需要修改缓存时间（单位：秒）。

### 重试配置

默认重试 1 次，可以在调用时修改：

```python
# 在 check_and_login_async() 函数中
login_result = await check_and_login_async(max_retries=2)  # 重试 2 次
```

## 注意事项

### ⚠️ 重要提示

1. **凭证安全**
   - 当前凭证硬编码在代码中
   - 生产环境建议使用环境变量
   - 不要将凭证提交到版本控制系统

2. **选择器稳定性**
   - 依赖 CSS 选择器和文本内容
   - 如果 FastMoss 网站改版，可能需要更新选择器
   - 选择器定义在 `LOGIN_SELECTORS` 字典中

3. **并发限制**
   - 同一时间只能有一个登录流程执行
   - 其他请求会等待登录完成
   - 这是正常行为，由 `asyncio.Lock` 保证

4. **缓存失效**
   - 缓存有效期 5 分钟
   - 超过 5 分钟会重新检查登录状态
   - 遇到 401 错误会自动标记为未登录

## 故障排查

### 问题：登录失败

**可能原因**：
1. 登录凭证错误
2. 网站登录流程改变
3. 网络连接问题

**解决方法**：
1. 检查凭证是否正确
2. 查看浏览器中的实际登录页面
3. 检查日志中的具体错误信息

### 问题：请求超时

**可能原因**：
1. 网络速度慢
2. 页面加载时间过长
3. 登录流程卡住

**解决方法**：
1. 增加超时时间
2. 检查网络连接
3. 手动在浏览器中测试登录

### 问题：并发请求全部失败

**可能原因**：
1. 登录锁异常
2. Playwright 未初始化

**解决方法**：
1. 重启 API 服务
2. 检查服务启动日志
3. 确保 Chrome 浏览器正常运行
