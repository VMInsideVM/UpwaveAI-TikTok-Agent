# 多窗口并发爬取功能指南

## 📖 概述

本指南介绍如何使用 Playwright API 的多窗口并发爬取功能,大幅提升达人详细数据获取速度。

**核心特性**:
- 使用多个独立浏览器窗口（BrowserContext）
- 所有窗口同时可见，实现真正的并行爬取
- 无需切换标签页，性能最优

## 🎯 性能提升

| 场景 | 顺序处理 | 3窗口并发 | 5窗口并发 |
|------|---------|----------|----------|
| 10个达人 | ~3分钟 | ~1分钟 | ~40秒 |
| 50个达人 | ~15分钟 | ~5分钟 | ~3分钟 |
| 100个达人 | ~30分钟 | ~10分钟 | ~6分钟 |
| 200个达人 | ~60分钟 | ~20分钟 | ~12分钟 |

**性能提升**: 3-5倍 ⚡

**为什么这么快？**
- 多个独立窗口同时运行，真正的并行处理
- 无需切换标签页，无额外开销
- 每个窗口独立执行点击和数据提取操作

## 🚀 快速开始

### 1. 确保 API 服务运行

```bash
# 启动 Playwright API 服务
python playwright_api.py
```

服务启动后访问 http://127.0.0.1:8000/docs 查看 API 文档。

### 2. 准备达人列表文件

通过 Agent 获取达人候选列表,会生成类似以下的 JSON 文件:

```
output/tiktok_达人推荐_女士香水_20250110_123456.json
```

### 3. 调用并发 API

**方式 A: 使用 curl**

```bash
curl -X POST "http://127.0.0.1:8000/process_influencer_list_concurrent" \
  -H "Content-Type: application/json" \
  -d '{
    "json_file_path": "output/tiktok_达人推荐_女士香水_20250110_123456.json",
    "cache_days": 3,
    "max_concurrent": 3
  }'
```

**方式 B: 使用 Python**

```python
import requests

response = requests.post(
    "http://127.0.0.1:8000/process_influencer_list_concurrent",
    json={
        "json_file_path": "output/tiktok_达人推荐_女士香水_20250110_123456.json",
        "cache_days": 3,
        "max_concurrent": 3
    },
    timeout=1800
)

result = response.json()

if result["success"]:
    print(f"✅ 处理完成!")
    print(f"   总数: {result['total_count']}")
    print(f"   缓存: {result['cached_count']}")
    print(f"   新获取: {result['fetched_count']}")
    print(f"   失败: {result['failed_count']}")
    print(f"   耗时: {result['elapsed_time']}")
else:
    print(f"❌ 处理失败: {result['message']}")
```

**方式 C: 使用测试脚本**

```bash
python test_concurrent_scraping.py
```

测试脚本会自动对比顺序处理和并发处理的性能。

## ⚙️ 参数说明

### json_file_path
- **类型**: string
- **必填**: 是
- **说明**: 达人列表 JSON 文件的路径
- **示例**: `"output/tiktok_达人推荐_女士香水_20250110_123456.json"`

### cache_days
- **类型**: integer
- **必填**: 否
- **默认值**: 3
- **范围**: 1-30
- **说明**: 缓存有效天数,在有效期内的数据不会重复爬取

### max_concurrent
- **类型**: integer
- **必填**: 否
- **默认值**: 3
- **范围**: 1-10
- **说明**: 最大并发窗口数
- **建议**:
  - `3`: 平衡性能与稳定性(推荐) - 3个独立窗口同时运行
  - `5`: 最大化速度,适合大批量处理 - 5个独立窗口同时运行
  - `>5`: 不建议,可能触发反爬或资源不足

## 📊 返回结果

```json
{
  "success": true,
  "total_count": 100,
  "cached_count": 20,
  "fetched_count": 75,
  "failed_count": 5,
  "failed_ids": ["7123456789", "7234567890"],
  "elapsed_time": "0:09:45",
  "message": "处理完成：20 个使用缓存，75 个重新获取，5 个失败"
}
```

### 字段说明

- `success`: 是否成功
- `total_count`: 总达人数
- `cached_count`: 使用缓存的数量(跳过爬取)
- `fetched_count`: 重新爬取的数量
- `failed_count`: 失败的数量
- `failed_ids`: 失败的达人 ID 列表
- `elapsed_time`: 总耗时
- `message`: 处理结果摘要

## 🔧 工作原理

### 并发流程

```
1. 读取达人列表 JSON 文件
   ↓
2. 检查缓存,过滤需要爬取的 ID
   ↓
3. 创建 N 个独立浏览器窗口（BrowserContext）
   ↓
4. 使用信号量控制并发数
   ↓
5. 所有窗口并行爬取达人详情（真正并行）
   ↓
6. 关闭所有浏览器窗口
   ↓
7. 返回统计结果
```

### 技术特性

1. **真正并行（Multi-Context 架构）**
   - 每个窗口 = 独立的 BrowserContext
   - 所有窗口同时可见，可同时执行点击操作
   - 无需 `bring_to_front()` 切换标签页
   - 理论提速接近并发数倍

2. **智能缓存**
   - 自动检查 `capture_time` 字段
   - 跳过未过期的数据
   - 只爬取缺失或过期的达人

3. **并发控制**
   - 使用 `asyncio.Semaphore` 限制并发数
   - 避免过度并发导致的问题
   - 自动分配任务到空闲窗口

4. **容错机制**
   - 单个达人失败不影响其他任务
   - 自动记录失败的达人 ID
   - 支持后续重试

5. **实时进度**
   - 每 10% 显示进度
   - 显示成功/失败统计
   - 显示预估剩余时间

## ⚠️ 注意事项

### 1. 反爬风险
- 并发数过高可能触发网站反爬机制
- 建议并发数不超过 5 个窗口
- 如遇到频繁失败,降低并发数

### 2. 系统资源
- 每个独立窗口（Context）占用 100-200MB 内存
- 5个窗口约需 1GB 内存
- 确保系统有足够的可用内存
- 注意：多窗口会同时显示在屏幕上

### 3. 网络稳定性
- 并发模式对网络稳定性要求更高
- 网络不稳定时建议使用顺序处理
- 可通过减少并发数降低网络压力

### 4. 浏览器状态
- 确保 Chrome 浏览器正常运行
- 确保 CDP 端口 9224 可访问
- 避免手动操作正在爬取的浏览器

## 🆚 对比:顺序 vs 并发

### 顺序处理 (原方式)

**端点**: `POST /process_influencer_list`

```python
# 不支持并发,按顺序逐个处理
requests.post(
    "http://127.0.0.1:8000/process_influencer_list",
    json={
        "json_file_path": "output/your_file.json",
        "cache_days": 3
    }
)
```

**适用场景**:
- ✅ 达人数量较少(<20个)
- ✅ 网络不稳定
- ✅ 对速度要求不高

### 并发处理 (新方式)

**端点**: `POST /process_influencer_list_concurrent`

```python
# 支持多标签页并发
requests.post(
    "http://127.0.0.1:8000/process_influencer_list_concurrent",
    json={
        "json_file_path": "output/your_file.json",
        "cache_days": 3,
        "max_concurrent": 3  # 关键参数
    }
)
```

**适用场景**:
- ✅ 达人数量较多(>20个)
- ✅ 网络稳定
- ✅ 需要快速获取数据

## 🐛 故障排查

### 问题 1: 连接 API 失败

**错误**: `❌ 无法连接到 Playwright API 服务`

**解决**:
```bash
# 确认 API 服务正在运行
python playwright_api.py
```

### 问题 2: 文件不存在

**错误**: `文件不存在: output/xxx.json`

**解决**:
- 检查文件路径是否正确
- 确保文件存在于 output 目录

### 问题 3: 大量失败

**错误**: `failed_count` 很高

**解决**:
- 降低并发数 (从 5 降到 3)
- 检查网络连接
- 查看 API 服务日志

### 问题 4: 浏览器资源不足

**错误**: 浏览器崩溃或卡死

**解决**:
- 降低并发数
- 关闭其他占用内存的程序
- 重启 Chrome 浏览器

## 💡 最佳实践

1. **首次使用**: 从 `max_concurrent=3` 开始测试
2. **大批量处理**: 达人数 >50 时使用并发模式
3. **网络不稳定**: 降低并发数或使用顺序处理
4. **定期清理**: 定期清理过期缓存释放磁盘空间
5. **监控日志**: 关注 API 服务日志,及时发现问题
