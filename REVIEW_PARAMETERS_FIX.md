# Review Parameters 强制转发修复

## 问题描述

**现象**：Agent 有时会跳过显示参数确认，或者总结/改写参数内容，而不是完整展示给用户。

**用户期望**：
```
📋 **当前筛选参数摘要**

🎯 **商品信息**
   • 商品名称: 女士香水
   • 商品分类: 美妆个护 > 香水彩妆 > 女士香水
   • 目标数量: 50 个达人

🌍 **目标地区**: 美国

🔍 **筛选条件**
   • 粉丝数: 10万 - 25万
   • 推广渠道: 不限制
   • 联盟达人: 不限制
   ...

---

请确认以上参数是否满意？
• 如果满意，请回复：好的/确认/可以/开始
• 如果需要调整，请告诉我要修改哪些参数
```

**实际情况（错误）**：
- ❌ 有时跳过不显示
- ❌ 或者总结成："好的，我已经为您整理了筛选参数：商品女士香水，粉丝10万-25万..."
- ❌ 或者改写成："参数设置完成，请确认..."

---

## 问题根因

Agent 使用 LLM 进行推理，即使有明确的指令，也可能：
1. **忘记调用** `review_parameters` 工具
2. **调用了，但总结工具返回值**而不是完整转发
3. **添加额外的解释**而不是直接输出

这是 LLM 的自然倾向：它倾向于"理解并总结"而不是"机械转发"。

---

## 解决方案

采用**三层强化策略**确保 Agent 必须完整转发参数：

### 1. 工具描述中添加强制要求

**文件**：`agent_tools.py:653-654`

```python
description: str = """
...
⚠️ 【强制要求】调用此工具后，你**必须**将返回的完整文本逐字逐句地转发给用户，
不得进行任何总结、省略、改写或添加额外内容！这是系统强制要求，违反将导致流程错误！
"""
```

### 2. 流程步骤中详细说明

**文件**：`agent.py:121-139`

添加了**【强制转发规则】**详细说明：

```
- **【强制转发规则】调用 review_parameters 后的处理流程**:
  1. 工具会返回格式化的参数摘要（包含 emoji、标题、所有筛选条件）
  2. **你必须将工具返回的文本完整地、逐字逐句地复制到你的回复中**
  3. **严禁进行任何形式的总结、省略、改写或添加内容**
  4. **不要说"参数已展示"、"请确认参数"之类的额外话**
  5. **直接将工具返回的文本作为你的完整回复发送给用户**
```

并提供了**正确示例**和**错误示例**对比。

### 3. 重要规则中设为最高优先级

**文件**：`agent.py:257-262`

在"重要规则"部分添加了专门的规则：

```
- **⚠️ review_parameters 工具的特殊规则（最高优先级）**:
  * 调用 review_parameters 后，**必须将工具返回的文本完整复制到你的回复中**
  * **严禁总结、改写、省略**工具返回的任何内容
  * **不要添加**"参数已展示"、"请确认"等额外话语
  * **直接转发**工具返回的完整文本即可
  * 这是强制性规则，违反会导致严重的用户体验问题
```

---

## 强化策略详解

### 为什么需要三层强化？

**单层不够**：
- 只在工具描述中说明 → Agent 可能"忘记"或"认为总结更好"
- 只在流程中说明 → Agent 可能"没仔细读"或"理解错误"
- 只在规则中说明 → Agent 可能"被其他规则覆盖"

**三层组合**：
- **工具描述** → Agent 调用工具时看到警告
- **流程步骤** → Agent 执行流程时看到详细指令
- **重要规则** → Agent 每次推理时都会参考的核心规则

通过**多次重复**和**不同位置强调**，大幅提高 Agent 遵守规则的概率。

## 技术细节

### 为什么不能自动强制？

**理想方案**：在代码层面强制 Agent 输出工具返回值
- 例如：`if tool == "review_parameters": force_output(tool_result)`

**现实限制**：
- LangChain ReAct Agent 的输出是由 LLM 生成的，不是代码控制的
- Agent 会读取工具返回值，然后**自主决定**如何回复用户
- 我们无法直接控制 Agent 的输出，只能通过提示词引导

**最佳实践**：
- ✅ 通过**多层强化的提示词**增加遵守概率
- ✅ 在工具、流程、规则三处重复强调
- ✅ 提供明确的正确/错误示例
- ✅ 使用"强制"、"必须"、"严禁"等强烈词汇

---

## 后续改进空间

如果 Agent 仍然偶尔不遵守，可以考虑：

1. **后处理钩子**：
   - 检测 Agent 输出中是否包含完整的参数摘要
   - 如果缺失，自动追加工具返回值

2. **自定义 Agent 类**：
   - 继承 LangChain Agent
   - 重写输出逻辑，强制转发特定工具的返回值

3. **用户反馈循环**：
   - 添加"参数未显示？点击查看"按钮
   - 记录 Agent 不遵守规则的案例，定期分析
