# 报告标题与会话标题同步修复

## 问题描述

之前报告库和聊天历史的标题不一致:
- **会话标题**: 通过 LLM 智能生成(基于对话历史)
- **报告标题**: 硬编码为 `{product_name} - 达人推荐报告`

这导致用户看到报告库和聊天历史的标题不匹配。

## 修复方案

修改报告创建逻辑，使报告标题自动使用对应会话的标题。

### 修改文件

#### 1. `background_tasks.py` - `_create_report()` 方法

**修改位置**: 第 504-555 行

**修改内容**:
```python
# 旧逻辑 (行 535)
title=f"{product_name} - 达人推荐报告",

# 新逻辑 (行 519-532, 550)
# 获取会话信息
session = db.query(ChatSession).filter(
    ChatSession.session_id == session_id
).first()

# 使用会话标题作为报告标题（如果是"新对话"则使用产品名称）
if session.title and session.title != "新对话":
    report_title = session.title
else:
    report_title = f"{product_name} - 达人推荐报告"

# 创建报告时使用会话标题
title=report_title,
```

#### 2. `api/reports.py` - `generate_report()` 端点

**修改位置**: 第 113-144 行

**修改内容**:

## 逻辑说明

### 标题选择规则

1. **优先使用会话标题**: 如果会话已经有智能生成的标题(不是"新对话")，则报告使用该标题
2. **降级使用产品名称**: 如果会话标题仍是"新对话"，则使用 `{product_name} - 达人推荐报告`

### 会话标题生成时机

会话标题通过 `session_manager_db.py` 的 `auto_update_title_if_needed()` 方法自动生成:

- **触发条件**:
  - 当前标题是"新对话"
  - 会话有至少 4 条消息(2轮对话)

- **生成方式**:
  - 使用 LLM 基于对话历史生成(5-15个字)
  - 自动处理重复标题(添加序号)

### 标题同步流程

```
用户开始对话
    ↓
创建会话(标题="新对话")
    ↓
用户发送消息 → 助手回复
    ↓
第2轮对话后，自动生成智能标题 (例: "推广美国女士香水")
    ↓
用户请求生成报告
    ↓
创建报告，使用会话标题 ✅
    ↓
报告库和聊天历史显示相同标题 ✅
```

## 优势

1. **用户体验更好**: 报告库和聊天历史标题一致，易于识别和管理
2. **标题更智能**: 利用已有的 LLM 生成的智能标题，不需要重复生成
3. **自动处理重复**: 会话标题已经处理了重复情况(添加序号)
4. **向后兼容**: 如果会话标题未生成(仍是"新对话")，则使用产品名称作为降级方案

## 测试场景

### 场景 1: 正常流程(会话标题已生成)

1. 用户开始对话，会话标题="新对话"
2. 用户发送2轮对话
3. 系统自动生成会话标题="推广美国女士香水"
4. 用户请求生成报告
5. **期望结果**: 报告标题="推广美国女士香水"

### 场景 2: 快速生成报告(会话标题未生成)

1. 用户开始对话，会话标题="新对话"
2. 用户只发送1条消息(对话不足2轮)
3. 系统未自动生成会话标题
4. 用户请求生成报告
5. **期望结果**: 报告标题="{product_name} - 达人推荐报告"

### 场景 3: 重复标题处理

1. 用户创建会话A，标题="推广女士香水"
2. 用户创建会话B，基础标题也是"推广女士香水"
3. 系统自动将会话B标题改为"推广女士香水 (2)"
4. 两个会话分别生成报告
5. **期望结果**:
   - 会话A的报告标题="推广女士香水"
   - 会话B的报告标题="推广女士香水 (2)"

## 注意事项

1. **时机问题**: 确保在生成报告时，会话标题已经生成。如果用户在对话很早期就生成报告，可能会使用降级标题。

2. **标题更新**: 如果用户手动修改会话标题，已生成的报告标题**不会**自动更新。这是合理的，因为报告是历史快照。

3. **标题长度**: 会话标题已经限制在 5-30 个字，适合作为报告标题。
