# 管理后台新功能快速上手指南

## 🚀 开始使用前

### 1. 运行数据库迁移（必须！）

**Windows**:
```bash
sqlite3 chatbot.db < migrations\002_credit_history.sql
```

**Linux/macOS**:
```bash
sqlite3 chatbot.db < migrations/002_credit_history.sql
```

### 2. 重启服务
```bash
# 重启聊天机器人服务
python start_chatbot.py
```

## 📊 新功能一览

### 1️⃣ 查看用户积分历史

**入口**: 用户管理 → 点击"积分历史"按钮

**显示内容**:
- 📅 变动时间
- 🏷️ 变动类型（充值/扣除/退还）
- 💰 变动积分（绿色=增加，红色=减少）
- 📈 变动前后积分
- 📝 变动原因

**用途**: 审计用户积分变动，追踪异常操作

---

### 2️⃣ 查看用户聊天记录

**入口**: 用户管理 → 点击用户名

**第一步 - 会话列表**:
- 显示该用户的所有聊天会话
- 会话标题、创建时间、消息数量

**第二步 - 消息详情**:
- 点击会话标题查看完整对话
- 用户消息（灰色背景）
- AI助手消息（淡蓝色背景）

**用途**: 了解用户使用情况，排查问题对话

---

### 3️⃣ 数据排序

**可排序列**:
- 总积分 ⬆️⬇️
- 已使用积分 ⬆️⬇️
- 剩余积分 ⬆️⬇️
- 总聊天数 ⬆️⬇️（新）
- 总Token数 ⬆️⬇️（新）

**使用方法**:
- 点击列标题一次：升序排列
- 再点击一次：降序排列
- 当前排序列会显示箭头

**用途**: 快速找到高活跃用户、积分不足用户等

---

### 4️⃣ 总聊天数和总Token数

**总聊天数**: 用户创建的所有会话数量
**总Token数**: 基于已使用积分估算（用于成本分析）

**用途**: 评估用户活跃度和系统成本

---

## 💡 使用技巧

### 快速找到高消耗用户
1. 点击"总Token数"列标题
2. 降序排列（大箭头向下）
3. 最上面的就是消耗最多的用户

### 检查用户积分异常
1. 点击"积分历史"按钮
2. 查看最近的变动记录
3. 检查是否有异常的充值或扣除

### 查看问题对话
1. 点击用户名
2. 查看最近的会话列表
3. 点击会话标题查看完整对话
4. 定位问题所在的消息

## 🎨 界面说明

### 积分变动颜色
- 🟢 绿色数字 = 积分增加
- 🔴 红色数字 = 积分减少

### 变动类型徽章
- 🟢 充值（add）= 绿色徽章
- 🟡 扣除（deduct）= 黄色徽章
- 🔵 退还（refund）= 蓝色徽章
- 🔵 调整（adjust）= 蓝色徽章

### 消息气泡颜色
- 👤 用户消息 = 灰色背景 + 蓝色左边框
- 🤖 AI助手消息 = 淡蓝色背景 + 绿色左边框

## ⚠️ 注意事项

1. **Token数是估算值**
   - 公式: `已使用积分 × 10`
   - 仅供参考，不是真实Token数

2. **积分历史默认显示最近100条**
   - 如需查看更多，请联系技术人员调整

3. **排序仅对当前页面有效**
   - 如果用户总数超过100，部分用户不参与排序

## 🐛 常见问题

### Q: 点击"积分历史"没有反应？
A: 检查数据库迁移是否已运行，确认 `credit_history` 表存在

### Q: 总聊天数显示为0？
A: 该用户确实没有创建过聊天会话，或会话已被删除

### Q: 总Token数不准确？
A: Token数是估算值，仅供成本分析参考

### Q: 点击用户名没有弹窗？
A: 检查浏览器控制台是否有错误，可能是API调用失败
